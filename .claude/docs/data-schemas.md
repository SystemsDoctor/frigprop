# Data schemas (so you never need to open `tables/*.json`)

All generated by `scripts/generate_tables.py` (CoolProp 7.2.0). One directory
per fluid key (`R134a`, `R744`, …) containing `sat.json`, `superheat.json`,
`subcool.json`, plus a top-level `tables/manifest.json`.

> PLAN.md Phase 1 replaces the absolute-T single-phase grids with
> ΔT-from-saturation grids. Update this doc when the schema changes and bump
> `schema_version` in the files + `tables.js` assertion.

## tables/manifest.json

```json
{
  "generated_utc": "...", "coolprop_version": "7.2.0",
  "fluids": {
    "R134a": { "files": [...], "T_crit_C": 101.06, "P_crit_kPa": 4059.3,
                "T_min_C": -50, "T_max_C": 100, "P_max_kPa": 5000 }
  }
}
```
14 fluids: R32, R1234yf, R1234ze, R454B, R452B, R134a, R410A, R407C, R404A,
R22, R290, R600a, R744, R717.

## sat.json — saturation table

- `columns`: `["T","P_sat","hf","hg","sf","sg","rhof","rhog"]`
- `rows`: arrays in column order, T ascending in 0.5 °C steps (0.1 °C near
  critical for R744), ending at min(T_max, T_crit − 0.5).
- Units: °C, kPa, kJ/kg, kJ/kg·K, kg/m³ (same convention everywhere).

## superheat.json / subcool.json — single-phase grids

```json
{
  "fluid": "R134a", "type": "superheat",
  "T_values_C": [-20, -10, ...], "P_values_kPa": [50, 75, ...],
  "properties": ["h", "s", "rho", "cp"],
  "grid": { "h": [[...]], "s": [[...]], "rho": [[...]], "cp": [[...]] }
}
```
- `grid[prop][i][j]` = value at `T_values_C[i]`, `P_values_kPa[j]`.
- Cells are `null` where the state is on the wrong side of saturation
  (within ±0.5 K). This null-handling is the root cause of PLAN.md bugs A1–A4.

## data/refrigerants.json — metadata (hand-maintained, editable)

Per fluid: `ashrae_designation, chemical_name, formula, molecular_weight,
safety_class, GWP_AR4, GWP_AR5, ODP, normal_boiling_point_C, T_crit_C,
P_crit_kPa, T_glide_C, blend_components, refrigerant_type, regulatory_status,
typical_applications[], replaces[], replaced_by[]`.
Keys here must match `tables/manifest.json` fluid keys exactly.

## Inspecting tables without reading them

```bash
python3 -c "import json; t=json.load(open('tables/R134a/superheat.json')); \
print(t['T_values_C'], t['P_values_kPa'])"
```

## Backend input pairs (`tables.js getProps`)

`TQ` (T °C, quality) · `PQ` (P kPa, quality) · `TP` (single-phase) ·
`PH` (P, h) · `PS` (P, s). Returns `{T_C, P_kPa, h, s, rho, x}` (+`cp` when
available); `x` is `null` for single-phase states.
